> 최종 업데이트: 2026-03-27

---

## 시스템 구조

```
[고객 음성/터치] → [STT] → [KT 봇프레임워크(SOE)] → [LLM Agent(NLU/RAG)] → [효성 클라이언트(UI)]
```

| 역할 | 담당 | 설명 |
|------|------|------|
| SOE/오케스트레이션 | KT | 봇프레임워크, 시나리오 제어, 분기 처리 |
| NLU/RAG/답변생성 | 초거대AI (자사) | LLM 에이전트, 개체추출, QA, 태스크처리 API |
| 클라이언트/UI | 효성 | 화면 렌더링, 컴포넌트형 템플릿 |
| 인프라 | 신한DS | 운영 도급, 형상관리, EAI |

## 기술 요구사항

### LLM Agent

- 개체 추출 (슬롯필링) - 화면 기반 + 음성 기반
- QA: RAG (에스베이식, 헬프데스크, 솔루션 데이터소스)
- 시나리오 복귀 / 다른업무 이동 (5개 업무)
- TTS 출력을 위한 문장 단위 스트리밍
- 컨텍스트 오류 시 히스토리 제외 처리

### 데이터

- 데이터소스: 에스베이식(업무기준), 헬프데스크(유저QA), 솔루션
- 청킹: size 250, overlap 100, 구분자 `###`
- 일배치: J-Flow 통해 매일 새벽 5시 자동 처리
- 핫픽스: 112 서버 직접 접속 → MD 수정 → 주피터 노트북 실행

### 인프라

- Docker → Kubernetes 전환
- 형상관리, EAI 연동 (ITBPI 프로젝트)
- 기기별 다른 모델 서빙 가능 여부 검토

## 주요 제약사항

1. **CDD/EDD**: AI 통한 고객확인 시 책임소재 → 비대면 채널과 동일 수준 절차 + 영업점 후선 확인
2. **비대면 실명인증**: 화상상담 시 인증 프로세스 부재 → 전담 직원 배치(단기), 혁신금융서비스(중장기)
3. **컨시어지 딜레마**: 없으면 사용 불가, 있으면 AI 몰입 저해
4. **디자이너 부재**: AI데스크 프로젝트 종료 후 철수, 피그마/스토리보드 업데이트 리소스 부족
5. **운영 도급**: 신한DS 기존 인력으로 충당 가능 여부 → 볼륨 파악 필요
