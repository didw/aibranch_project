# 프로젝트 타임라인

> AI 브랜치 AICC 프로젝트 주요 이벤트 (Notion 회의록 기반)

---

## 2025

### 2월 - 프로젝트 킥오프

- 데이터 파이프라인 구성 논의 (화상 데이터 → 학습 데이터)
- LLM 목적 정의: 예외상황 해소, 음성 기반 업무 구분, 인터페이스 단축
- LLM 에이전트 프로토타입 제작 (워크샵 대비)
- 이슈: LLM 버벅임, STT 숫자인식 오류, 한도제한계좌 문의 대응

### 3월 - 개발 본격화

- **아키텍처**: UX-LLM 인터페이스 협의, 화면기반 개체추출 설계
- **시나리오 기획**: 음성 진행 + 시나리오 + QA, 다른업무 이동 로직 (60여개)
- **개발**: 화면정보 screen_data 전달, 계좌 선택/복수 개체 추출
- **플랫폼**: 형상관리, EAI 연동, 데이터 배치 구조 논의
- 일정변경: 5월 1주차 오픈 → 이후 재조정
- 워크샵(3/5): 컨시어지 모델, 화상상담 데이터 활용 논의

### 4월 - 개발/테스트

- **업무 개발 순서**: 통장사본 → 거래내역서 → 증명서 2종 → 체크카드
- 스토리보드 수정, 연호 지점 이동(4/14~)
- 효성 미팅: 씬/생성 로직 통합, 의도 분류, body 정답셋
- 형상관리: Docker → Kubernetes 전환 필요 (5월부터 가능)
- 이행 후 운영 프로세스 정립 논의
- 컨텍스트 오류 대응: 챗히스토리 제외 설계

### 5월 - 고도화 논의

- **고도화 방향**: KT 봇프레임워크 활용 방안, 응답속도 개선
- 업무 확대 범위 논의: 쉬운 업무 vs 어려운 업무 대체 전략
- 동시신규 업무 검토: 봇프레임워크 API 호출 가능 여부 확인

### 6월 - 설계 확정 및 QA

- **6/10 주간회의**: 완결률 78% (미완결 22%), AI 창구 업무 24개 추가 가능 분석
- **동시신규 프로세스 기획**: 본인확인 → LLM상담 → 가입처리 3단계
- **6/17**: 화상상담 데이터 활용 어려움 확인, CDD/EDD 이슈
- **6/18 KT 미팅**: 봇프레임워크 → 효성 클라이언트 화면 렌더링 구조 합의
- **6/24**: QA 개선 위한 지식관리 방안 (별도 게시판 개설)

### 7월 - AI Branch 2.0 설계

- **7/1 QA 인수인계**: 데이터소스 구조, 운영 프로세스, 핫픽스 절차 정리
- **7/8 주간회의**: AI 창구 운영 현황, 컨시어지 딜레마, 지점 과제 부여
- **7/9 KT 시연**: 봇프레임워크-LLM Agent 연계 데모
  - **합의**: 특정 의도만 봇으로 연계, QA는 자체 처리
  - UI: 컴포넌트형 템플릿 동적 조합 방식 확정
  - 역할: 효성/초거대AI = 태스크처리 API, KT = 시나리오 오케스트레이션
- AI Branch 2.0 범위: 동시발급 (입출금, 체크, 인터넷뱅킹), 8월 이행 목표

## 2026

- 게이트웨이 저장소, 전문 26건 운영계 반영 완료
- 컴플라이언스 문구 적용 작업 진행 중
