## 논의 사항

### 1. 현행 전세대출 프로세스 분석
- **페인 포인트**: 상품 복잡(기금+당행), 서류 안내 불명확, 고객 재방문 빈번, 종이 기반 처리
- **비대면 비중**: 카카오뱅크/토스뱅크 대비 당행 비중 압도적으로 낮음
- **SOL 프로세스**: 약 12단계, 정보 과도 요구 → 중도 이탈 빈번
- **경쟁사 비교**:
  - 카카오/토스: 대화형, 최소 정보로 예상 한도, 용어 쉬움
  - 당행: 첫 화면부터 제한사항, 임대인 정보까지 입력 필요 (정확하지만 진입 장벽 높음)
  - 국민: 가능 상품군 제안 → 서류 접수 → 영업점/집중화센터 배분

### 2. 3대 개선 과제

#### 과제 1: 비대면 프로세스 개선 (SOL 앱)
- 12단계 → **대화형(챗봇)** 방식 전환
- 계약서 있는 경우: 스크래핑/AI OCR로 정보 자동 추출
- 계약서 없는 경우: 간이 한도 안내 분기 추가
- 고객이 상품 선택 → **시스템이 상품 추천** 구조로 변경
- 기존 SOL 유지 + **AI 전세대출 봇 별도 메뉴 신설** (병행)

#### 과제 2: 권리조사 자동화 및 검증
- 계약서 ↔ 등기부등본 AI 1차 점검
- 중개대상물 확인설명서로 OCR 교차 검증 (타이핑 데이터, 인식률 높음)
- 권리변동 알림 서비스 연계 (고솔 테크 임베디드 과제)
- 전세 안전도 점검 외부 업체 제휴 검토

#### 과제 3: 직원용/고객용 Q&A

### 3. AI 창구 사전 상담 방향
- **플로우**: 대출 종류 선택 → 기본 정보 입력(터치) → 서류 체크 → 미비 시 안내 / 완비 시 스캔 → 상담사 인계
- **민원 리스크 논의**:
  - "기계에 입력하라니" 불만 (고령층, 지방)
  - 계약서 없이 온 고객(60~70%) 돌려보내면 민원
  - KB 사례: DDC 중앙 배치 + 컨시어지 → 이용률 높음 / 당행: 구석에 방치
- **합의**: 서류 체크 앞단 배치 + 계약서 없는 고객에게 **간이 한도(레인지) 제공**
