## 논의 사항

### 1. 전세대출 PI 배경

### 2. AI 브랜치 PoC 현황
- **시나리오**: 고객이 상품을 알고 온 경우 / 모르는 경우 2가지
- **플로우**: 기본 정보(소득/나이/가족/주택보유) → 기금 4종(버팀목 등) 자격 판단 → 해당 없으면 당행 전세 → 금리/한도 개략 안내
- **구현 방식**: 프로세스 로직으로 분기, LLM은 대화 자연스러움과 슬롯 추출에 활용
- **디금센 핵심 요구**: 대출 불가 고객 **사전 필터링** + 1차 상담 내역 정리 후 직원 전달

### 3. 대화형 vs 버튼형 UI 방식
- 자유 대화형: 복잡 케이스 대응 불가, LLM 오답 시 민원 리스크
- **결론**:
  - 앞단(불가 필터링): **버튼식 강제** (주택보유, 소득초과 등)
  - 뒷단(상담): LLM 대화형으로 상세 상담

### 4. 한도/금리 정확도 향상
- 현재: 보증금 80% 단순 계산 → 매우 부정확
- 향상 방안: NICE 신용평가(AS등급), 보증기관(HF/HUG/SGI) 한도 조회
- 기술 제약: 슈퍼SOL은 D서비스, AI 브랜치는 G/W → **MCP 툴** 개발 요청 필요
- **기준**: 인터넷전문은행 수준으로 잡고 개발부 확인

### 5. QA 지식 기반 정비
- 고객용: 슈퍼SOL 탑재 (전세대출 제도/정책 + 상품설명서)

- 정부 정책 연간 40회+ 변경 → 최신화가 핵심

### 6. 권리 조사 검증 (고객솔루션부 요구)
- 권리 조사서 + 고객 입력 + 서류를 대사(cross-check)하여 누락/오류 검증
- 직원 면책/리스크 제로화 목적
